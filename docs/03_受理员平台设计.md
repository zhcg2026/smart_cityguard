# 受理员平台设计（讨论中）

## 登录方式
- APP端 + Web端

## 案件来源渠道
| 渠道 | 录入方式 |
|------|----------|
| 领导交办 | 系统下发 |
| 采集员采集 | APP上报 |
| 视频监控 | 智能识别对接 |
| 12345热线 | 受理员登记 |
| 市民投诉 | 受理员登记 |

## 角色定位
- 受理员 ←→ 问题来源（隔离于处置部门）
- 案件受理后批转给派遣员，不直接对接处置部门

---

## 受理员栏目（待确认）

| 栏目 | 功能说明（初步理解） |
|------|---------------------|
| 登记栏 | 手动登记12345热线、市民投诉等案件 |
| 待查栏 | 无照片/视频的案件，等待核实 |
| 立案栏 | 信息完整的案件，待批转或作废 |
| 核查栏 | 已处置案件，等待核查结果 |
| 经办案件 | 已处理的案件历史记录 |
| 不受理栏 | 作废案件记录 |

---

## 登记内容

### 举报人信息
- 接入电话
- 来电时间
- 姓名
- 是否公开
- 是否回访
- 性别

### 案件信息（必填）
- 问题描述
- 位置（可地图标记）
- 问题来源（可选）
- 问题等级：日常 / 一般 / 严重 / 重大
- 案件类型：默认城市管理类
- 问题类型：事件 / 部件 / 服务事项
- 大类、小类
- 立案条件（可选）
- 处置时限

### 案件信息（非必填）
- 附件（照片、视频、录音）
- 区域（某某县、某某区）
- 街道（某某办事处）
- 社区（某某社区）
- 采集员（可选）
