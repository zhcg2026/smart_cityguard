# 采集端套壳 Android — Capacitor 方案

## 前提

- 技术栈：Vue 3 + Vite 5 + Vant 4 + Pinia + AMap
- 平台：仅 Android
- 原生功能：摄像头拍照、GPS 定位、推送通知、相册选择

---

## Phase 1 — 基础搭建（约 2 小时）

### 1.1 清理 uni-app 遗留文件

项目根目录有已废弃的 uni-app 代码（App.vue、main.js、pages.json、manifest.json、uni.scss、pages/、static/、store/），与 `src/` 下的活跃代码完全重复，直接删除。

### 1.2 安装 Capacitor 并初始化

```bash
cd D:\smart_cityguard\frontend\collector-app
npm install @capacitor/core @capacitor/cli @capacitor/android
npx cap init "智慧城管采集" "com.smartcityguard.collector" --web-dir dist
npx cap add android
```

### 1.3 路由改为 Hash 模式（关键）

Capacitor 在 Android 上以 `file://` 协议加载，`createWebHistory` 无法工作，必须改用 hash 模式。

**文件**：`src/router/index.js:70`
```js
// createWebHistory() → createWebHashHistory()
```

### 1.4 API 地址环境变量化

生产环境无 Vite 代理，需通过环境变量配置后端地址。

**文件**：`src/utils/request.js:7` — `baseURL` 改为读取 `import.meta.env.VITE_API_BASE || '/api'`
**文件**：`src/utils/fileUrl.js:18` — 同理，硬编码的 `/api` 需替换
**新建**：`.env.production`、`.env.android-dev`

---

## Phase 2 — 原生功能集成（约 6 小时）

### 2.1 GPS 定位

安装：`@capacitor/geolocation`

**关键问题**：AMap 用 GCJ-02 坐标，原生 GPS 返回 WGS-84，偏差约 100-500 米。需新建 `src/utils/coordTransform.js` 做 WGS-84 → GCJ-02 转换。

**文件**：`src/views/report/index.vue` — 更新 `locateOnMap()`，用 Capacitor Geolocation 替代 AMap Geolocation 插件，转换坐标后注入地图。

### 2.2 摄像头拍照 + 相册选择

安装：`@capacitor/camera`

**新建**：`src/utils/cameraHelper.js` — 封装 `Camera.getPhoto()` 和 `Camera.pickImages()`，返回 File 对象兼容现有 `uploadFile()` API。

**文件**：`src/views/report/index.vue` — 替换 `van-uploader` 的拍照触发逻辑。

### 2.3 推送通知

安装：`@capacitor/push-notifications`

- 前台：用 Vant Notify 显示
- 后台/离线：FCM 系统通知
- App 切后台时暂停轮询，切前台恢复

**文件**：`src/App.vue` — 添加推送初始化 + App 状态监听

**后端要求**：需新增 FCM token 注册接口 + Firebase Admin SDK 推送服务。

### 2.4 其他插件

| 插件 | 用途 |
|---|---|
| `@capacitor/app` | 前后台状态检测 |
| `@capacitor/keyboard` | 键盘弹出适配 |
| `@capacitor/status-bar` | 状态栏样式 |
| `@capacitor/preferences` | 可选，替代 localStorage |

### 2.5 地图导航深链接

**文件**：`src/components/CaseLocationMap.vue:119` — `window.open` 改为 `App.openUrl({ url: 'amapuri://route/plan/...' })`

## Phase 3 — 推送服务搭建（约 4 小时）

1. Firebase Console 创建项目，下载 `google-services.json` 放入 `android/app/`
2. 配置 Android Gradle（google-services 插件 + firebase-bom）
3. 后端新增 FCM token 存储 + 推送发送接口

---

## Phase 4 — 签名发布 + 测试（约 4 小时）

### 构建流程

```bash
npm run build
npx cap sync android
# Android Studio 打开 → Build → Generate Signed Bundle/APK
```

### 关键测试项

| 测试 | 验证点 |
|---|---|
| 启动白屏 | Hash 路由是否生效 |
| 地图加载 | AMap WebView 兼容性 |
| GPS 定位 | 坐标转换精度 |
| 拍照上传 | 原生相机调用 + MinIO 上传 |
| 推送通知 | 前台/后台/杀进程三种状态 |
| Android 返回键 | 路由回退而非退出 App |
| 权限弹窗 | 相机/定位/通知权限流程 |
